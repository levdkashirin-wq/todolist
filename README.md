### Use Cases 
Основные 

1. Создать задачу
Пользователь может создать новую задачу, указав название (обязательно) и описание (опционально).
2. Просмотреть список задач  
Пользователь может получить список всех задач.
3. Просмотреть одну задачу
Пользователь может получить детальную информацию о задаче по её ID.
4. Редактировать задачу
Пользователь может изменить название и описание существующей задачи.
5. Изменить статус задачи
Пользователь может изменить статус выполнения. 
6. Удалить задачу
Пользователь может удалить задачу по ее ID.
7. Установить приоритет
Пользователь может задать приоритет задачи(LOW,MEDIUM,HIGH)
8. Установить срок
Пользователь может установить дату выполнения задачи.
9. Фильтрация по статусу
Пользователь может отфильтровать задачи по статусу(PENDING, IN_PROGRESS, DONE).
10. Сортировка по сроку
Пользователь может отсортировать задачи по дате выполнения.
### User Stories
1. Создание задачи
Как пользователь, я хочу создавать задачи, чтобы не забывать о важных делах.
2. Просмотр списка задач
Как пользователь, я хочу видеть список всех моих задач, чтобы быстро оценить объём работы.
3. Просмотр деталей задач
Как пользователь, я хочу смотреть детали конкретной задачи, чтобы вспомнить, что именно нужно сделать.
4. Редактирование
Как пользователь, я хочу редактировать задачу, чтобы исправить ошибку или изменить описание.
5. Изменение статуса(HIGH → MEDIUM → LOW).
Как пользователь, я хочу изменять статус задачи, чтобы отслеживать прогресс выполнения.
6. Удаление
Как пользователь, я хочу удалять задачу, если она больше не актуальна.
7. Приоритет 
Как пользователь, я хочу указывать приоритет, чтобы видеть, какие задачи важнее.
8. Срок выполнения
Как пользователь, я хочу указывать срок выполнения, чтобы ничего не просрочить.
9. Фильтрация
Как польз ователь, я хочу фильтровать задачи по статусу, чтобы фокусироваться на незавершённых делах.

![ER-диаграмма задач](docs/er-diagram.png)
Статусы выполнения (Task Status)

- PENDING - Задача создана, ожидает старта.
- IN_PROGRESS - Задача в процессе выполнения.
- DONE - Задача выполнена.

  Приоритеты задач (Task Priority)

- HIGH - Высокий приоритет, нужно сделать срочно.
- MEDIUM -  Средний приоритет.
- LOW - Низкий приоритет, можно сделать позже.

Схема tasks
- id - SERIAL, PRIMARY KEY 
- title - VARCHAR(255), NOT NULL 
- description - TEXT 
- status - VARCHAR(50), NOT NULL, DEFAULT 'PENDING' 
- priority -VARCHAR(20), DEFAULT 'MEDIUM' 
- due_date - DATE 
- created_at - TIMESTAMP, DEFAULT CURRENT_TIMESTAMP 
- updated_at - TIMESTAMP, DEFAULT CURRENT_TIMESTAMP 
API Endpoints

GET /api/tasks
Получить все задачи. HTTP статусы: 200 OK

GET /api/tasks/{id}
Получить задачу по ID. HTTP статусы: 200 OK, 404 Not Found

POST /api/tasks
Создать новую задачу. HTTP статусы: 201 Created, 400 Bad Request

PUT /api/tasks/{id}
Обновить задачу. HTTP статусы: 200 OK, 404 Not Found

PATCH /api/tasks/{id}/status
Изменить статус задачи. HTTP статусы: 200 OK, 404 Not Found

DELETE /api/tasks/{id}
Удалить задачу. HTTP статусы: 204 No Content, 404 Not Found

Ошибки и HTTP статусы:
400 Bad Request — Некорректные данные запроса
Пример ответа:
{
"message": "Task title cannot be empty",
"statusCode": 400,
"timestamp": "2026-04-27T10:00:00"
}

404 Not Found — Задача не найдена
Пример ответа:
{
"message": "Task not found with id: 99",
"statusCode": 404,
"timestamp": "2026-04-27T10:00:00"
}

500 Internal Server Error — Внутренняя ошибка сервера
Пример ответа:
{
"message": "Internal server error",
"statusCode": 500,
"timestamp": "2026-04-27T10:00:00"
}
